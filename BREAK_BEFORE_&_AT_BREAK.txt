* AT BREAK IS NOT ONLY VALID FOR DATABASE FIELDS, IT IS ALSO VALID FOR USER DEFINED VARIABLES. 

* THE BEFORE BREAK PROCESSING COMMAND IS DESIGNED EXPRESSLY TO OVERCOME THE TIMING PROBLEM.

* IF YOU REMOVE THE "BEFORE BREAK PROCESSING" AND THE END-BEFORE, YOU WILL SEE THE DIFFERENCE IN THE OUTPUT.
*
DEFINE DATA LOCAL
1 EMP-VIEW VIEW OF EMPLOYEES
  2 NAME
  2 FIRST-NAME
  2 SALARY (1)
1 #INCREASED-SALARY (N6)
1 REDEFINE #INCREASED-SALARY
  2 #THOUSANDS (N3)
1 #LINE1 (A65)
1 #LINE2 (A65)
1 #COUNT (N5)
1 #OLD-COUNT   (N5)
END-DEFINE
*
READ (100) EMP-VIEW IN PHYSICAL SEQUENCE
END-ALL
SORT BY SALARY(1) USING NAME FIRST-NAME
*
BEFORE BREAK PROCESSING
       COMPUTE #INCREASED-SALARY = SALARY(1) * 1.1
       END-BEFORE
*
AT BREAK OF #THOUSANDS
   NEWPAGE
   MOVE OLD (#THOUSANDS) TO #OLD-COUNT
   MOVE COUNT (NAME) TO #COUNT
   COMPRESS 'THERE ARE' #COUNT 'PEOPLE WITH INCOME IN THE RANGE'
   INTO #LINE1
   COMPRESS #OLD-COUNT 'TO' #THOUSANDS 'THOUSANDS OF DOLLARS' INTO #LINE2
   WRITE 5/10 #LINE1 // 10T #LINE2
   NEWPAGE
   END-BREAK
*
DISPLAY NAME FIRST-NAME #INCREASED-SALARY
END-SORT
END
